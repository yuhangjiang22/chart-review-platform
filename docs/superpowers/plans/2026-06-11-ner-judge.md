# Increment 3.5 — wire up the NER LLM-judge for concur

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Make the LLM-as-judge actually adjudicate NER spans in concur. Today the
judge no-ops on `task_kind:"ner"` (`runJudgeBatch` only finds phenotype
field-assessment cells → `0/0 cells`). The `judgeSpan` surface was already copied
verbatim during the increment-1 judge port, but the batch/extractor/route/skill/UI
that drive it were not. Port them from v2 and apply the same deepagents
adaptations the phenotype judge needed.

**Program context:** Increment 3.5 of extending concur → v2 (judge ✅, NER ✅).
Port FROM v2 (`chart-review-platform-v2`), the proven reference. See
`2026-06-11-judge-phase.md` and `2026-06-11-ner-task-kind.md`.

**Design (from the v2 NER-judge study):**
- The NER judge's unit is a **span (or span-pair)**, not a (patient, criterion)
  cell. Its analog of "disagreement" is **span alignment**: two agents' spans are
  aligned and each contested one is classified `hard` / `soft` / `boundary` /
  `type_diff` / `miss` (plus single-agent `novel_candidate`). The alignment math
  lives in a pure package, `eval-span-iaa` (`computeSpanIaa`).
- **Two-agent requirement.** The judge only has work when agents disagree. NER
  agents diverge via `role_preset` (already threaded into both extract passes in
  concur's `direct-llm-extract.ts`). So e2e needs a **2-agent NER run with
  distinct presets** (e.g. `default` + `skeptical`).
- **Already in concur:** `judgeSpan` + `buildSpanJudgePrompt` + `JudgeSpanInput`/
  `JudgeSpanAnalysis` + `validateSpanAnalysis` (`server/lib/judge.ts:347-606`);
  the GET `/judge` endpoint (task-kind-agnostic, returns the on-disk file);
  rolePreset divergence in `pipeline-extract-ner`; the multi-agent run loop.
- **Must add:** `eval-span-iaa`; the NER section of `packages/disagreements`
  (`compareSpanDrafts`, `loadAgentSpanDrafts`); two deepagents fixes to
  `judgeSpan` (mcpServers + inline schema); the NER batch in `judge-batch.ts`
  (`runJudgeSpanBatch`, `buildSpanWorklist`); the route `task_kind` dispatch; the
  `chart-review-ner-judge` skill; the `PhaseJudge` NER branch (`NerJudgeCard`).
- **MVP scope — deferred (calibration, NOT judge-critical):**
  `ner-calibration-routes` + the F1/κ consumers, `NerCalibrationFigure`,
  `chart-review-ner-{calibrate,cohort,methods,codify}` skills. (`computeSpanIaa`'s
  F1/κ fields come along for free in J1 but have no judge-path consumer.)

**Reference (v2):** `packages/eval-span-iaa/src/index.ts`,
