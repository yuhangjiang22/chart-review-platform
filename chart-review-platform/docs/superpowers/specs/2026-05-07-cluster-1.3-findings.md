# Cluster 1.3 — overview_prose trajectory-residue hygiene (PS-5)

PS-5 from the persona stress test: when the reviewer pivots/reverts mid-build,
the build skill emits meta.yaml whose `overview_prose` carries a "reflection"
paragraph documenting the trajectory. This is reversion archaeology — confusing
to downstream readers, who want the rubric's documentation, not the chat history.

## Changes

1. **Schema** (`task-meta.schema.json`): added optional `overview_prose_check`
   enum["skip"] for the per-task escape hatch.

2. **Validator** (`build_skill_validator.py`): new heuristic over
   `meta.overview_prose` for trajectory phrases ("initially", "we revised",
   "first version", "we pivoted", "after revising", "scope drift", etc.).
   Diagnostic: `overview_prose_trajectory_residue` (warning).

3. **Build-skill prompt** (`SKILL.md`): new hard rule instructing the build
   skill to write clean overview_prose, with a pointer to
   `overview_prose_check: skip` for legitimate cases.

## Phrase list

The following 14 phrases are matched (case-insensitive, word-boundary anchored):

1. `initially`
2. `originally`
3. `first attempt`
4. `first version`
5. `first draft`
6. `first pass`
7. `we revised`
8. `we pivoted`
9. `we reverted`
10. `after revising`
11. `after reviewing`
12. `previously considered`
13. `scope drift`
14. `phase reversion`
