# Cluster 1.2.1 — heuristic tightening + skip backfill

Follow-up cluster after 1.2 surfaced a 90% false-positive rate (9/10) on the time-window discipline heuristic. Two changes:

1. **Section-scope the heuristic.** The validator now scans only the `prompt` field + content of `## Definition` and `## Extraction guidance` sections. Examples / Boundary / Failure-modes / other sections are excluded so situational prose stops triggering false warnings.
2. **Backfill `time_window_check: skip`** on the 7 criteria that remain false positives even after section-scoping. Inline comments document the rationale per criterion.

## Section-scoping detail

New helper `_extract_heuristic_body(body)` in `build_skill_validator.py`:

- Splits the body at level-2 headings (`^## .+$`).
- Keeps only blocks whose heading text is in `{"Definition", "Extraction guidance"}`.
- Discards intro prose before the first `##`, plus all other sections.
- The `prompt` frontmatter field is still scanned in full (no change).

Two new tests anchor the contract:
- `test_time_window_heuristic_scans_only_definition_and_extraction_guidance` — phrases in Examples / Boundary / Failure-modes don't trigger.
- `test_time_window_heuristic_still_fires_on_definition_section` — phrases in Definition still trigger (sanity).

Cluster 1.2's existing fixtures continue to fire correctly — both `at_index.md` and `history_of.md` have their trigger phrases in `## Definition`.
